Releases

What's new, improved, and fixed in TryAIToday Automator.

New to reference images and naming? See the Prompt Guide for copy-ready examples of @tag and #name.

Version 0.2.0

Latest

New

  • Reference an image by name — write @name in a prompt to pull that image in as a reference. Works with images named in Flow, uploaded this session, or made earlier in the same batch. Multi-word names use @[name with spaces].
  • Name your results with #name — add #name to a prompt to name the image it creates, then reuse it later with @name. Perfect for keeping the same character or product consistent across a whole batch.
  • Retry any job — regenerate a finished, failed, or stopped job as a new version (v2, v3…) with one click — even while the queue is still running (it jumps ahead and runs next).
  • Subqueues — when you retry a job whose result other jobs reference, start a subqueue to re-run that whole dependent chain in its own tab and subqueue_N folder, so every image stays in sync with the updated reference. Run several at once; all queues process round-robin.
  • Rerun, resume & regenerate — press Start on a finished batch to regenerate every image again as new versions in the same batch; on a stopped batch, choose to resume the remaining jobs (keeping what's already done) or start the whole batch over. Resumed jobs pick up any references you retried in the meantime.
  • Reference previews in the queue — every job shows a thumbnail of each @tag it uses before you start. A reference that a later prompt creates shows a "step N" chip that fills in with the real image once that step runs.
  • Click to preview — click any thumbnail (a generated result or a reference chip) to view it full-size instantly, no waiting for a refresh.
  • Edit a prompt — a ✎ button opens an editor to fix a wrong @name or a typo from a bulk load — before you start a run, or on a finished job right before retrying it.
  • Product tour — a quick first-run walkthrough of the whole panel, replayable anytime from Account.
  • New model: Nano Banana Lite — added alongside Nano Banana 2 and Nano Banana Pro.
  • More reference slots — Scene and Style now hold up to 5 images each (10 total across Subject + Scene + Style combined).
  • Fresh session between runs — the Flow tab refreshes automatically when a run ends or is stopped, so the next batch starts clean.
  • In-app Prompt Guide & support — a Prompt Guide button next to your prompts, plus a direct link to our Discord community for help.

Improved & fixed

  • Expired sign-in no longer shows a broken "Trial — / —" state — it takes you straight back to sign in.
  • "Unusual activity" recovery — a blocked generation is now auto-retried with a fresh security check, and a built-in guide walks you through clearing Google's block if it persists.
  • Clearer errors — daily-limit, rate-limit, and expired-session cases now show distinct, actionable messages.
  • Smoother queue — reference thumbnails no longer flicker when you press Start.
  • Trashed references — images you move to trash in Flow are no longer offered or used as @name references.
  • Retry downloads — retrying a job on a finished or stopped queue now saves into the batch folder instead of the Downloads root.
